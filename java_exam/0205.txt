Lambda & Anomymous
Lambda는 "메서드를 1개만 가진 인터페이스"에만 사용가능
@FunctionalInterface 애노테이션은 이를 강제함

PrintSomething funtion = (String message) -> {
	if (message == null) {
		System.out.println("파라미터 잘못 됨");
	} else {
		System.out.println(message.repeat(5));
	}
};
이렇게 사용하면 실제로 인터페이스의 메서드 인스턴스가 힙영역에 할당된다.

위에서 Lambda가 인터페이스에만 사용 가능한 이유는 추상클래스는 실제로 필드 값을
가질 수 있기 때문에, 강제성을 부여한 것임(항상 말하지만 적절한 제한이 있는 코드가 좋은 코드다)

Lambda로 메서드를 전달하면 그것은 힙 영역에 할당하지 않기 떄문.
